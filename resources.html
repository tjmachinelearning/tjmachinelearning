Recommended Courses

If you've read through our lectures and want to learn more, we recommend reading the lecture notes or watching the videos of the CS229 and CS231n courses. CS229 takes a more mathematical look at standard machine learning methods, while CS231n focuses on deep learning algorithms for visual processing.

In addition, if you need to brush up on your calculus knowledge, a good book is AOPS Calculus. The TJ BC Calculus material is also available to anyone for free on Blackboard. If you need work on your multivariable calculus knowledge, the TJ multi teachers use the Stewart Multivariable Calculus book, but Dr. Osborne's notes cover the much of the material in a more concise fashion.

Computational Resources

Unfortunately, TJ has a severe lack of GPUs. When the Computer Systems lab was upgraded a few years ago, the aging CPUs were replaced with then-modern i5-4690s. However, the GPUs remain GT 730s. If you would like to run your code on TJ machines, we recommend you use the computer called zoidberg, which is the only computer with a modern GPU. Simply ssh using the following commands:
ssh username@remote.tjhsst.edu
ssh zoidberg

Installing Libraries on TJ Linux Machines

All the necessary dependencies (CUDA, CuDNN) have been installed for Tensorflow, Keras, and PyTorch on the Linux machines. Due to user restrictions, each user must install the library themselves. Use these commands on zoidberg for Tensorflow and Keras, respectively:
pip3 install tensorflow-gpu --user
pip3 install keras --user

For PyTorch, check the latest commands on the official website. Select Python 2.7 and CUDA 8.0. You must use Python 2 for PyTorch on zoidberg because zoidberg has Python 2.7 and 3.4, but PyTorch supports Python 2.7, 3.5, and 3.6. The commands as of right now are:
pip install http://download.pytorch.org/whl/cu80/torch-0.1.12.post2-cp27-none-linux_x86_64.whl --user
pip install torchvision --user

If you are working on a Linux machine that is not zoidberg or duke (in other words, a standard workstation without powerful GPUs), you will need to install the CPU version of PyTorch. Simply go to the official site, select "Linux" for the OS, "pip" for the package manager, "3.5" or "2.7" depending on whether you want to use Python 3 or Python 2, and "None" for the CUDA version. Copy and paste the commands shown. Note, you will still need to add --user to all commands.

Installing Libraries on Your Own Linux Machine

If you want to install these libraries on your own Linux computer, just remove the --user for the above commands, since you presumably have root. If you have Python 3.5 on your machine, you can install Pytorch for Python3 like so:
pip3 install http://download.pytorch.org/whl/cu80/torch-0.2.0.post3-cp35-cp35m-manylinux1_x86_64.whl
pip3 install torchvision
For Python 3.6, replace every "35" with a "36" in the above commands. Check for your specific configuration on the official website. This assumes you have a CUDA 8.0 compatible GPU. If you don't, running any sort of deep learning model will take too long, and it is advisable you use the TJ GPU Cluster.

Python 2 or 3?

Machine Learning Club recommends Python 3 whenever possible. Almost all libraries have support for Python 3.5 and 3.6. Even though most support Python 2.7 as well, Python 2 support ends in 2020, so make the switch if you haven't already!
